DSL primitives — how each role behaves
send(v, A, B)
Role A — sender
Sign message · transport.send() · return sent value
Role B — receiver
Block until recv · verify ed25519 sig · return value
All others
Skip — line is invisible
choose(R, opts)
Role R — chooser
LLM call with evidence · broadcast result to ALL roles
All others — observers
Await broadcast · branch on received value identically
Guarantee
All roles see the same value — no divergence possible
gate(R, p, fn)
Role R — gate role
Call Gate Provider · await proof · log proof · broadcast result
All others
Skip / wait — cannot execute without proof
Guarantee
result.ok === true only if proof was issued
AppendLog — Secure Scuttlebutt-style hash chain
Each log entry
id,   kind,   role,   choreoId,   payload,   sig = ed25519(body, privkey),   prevId = SHA-256(prevEntry),   ts
Every entry is signed with the role's ed25519 private key.
prevId chains each entry to the last — any tampering breaks the hash chain.
Snapshot uploaded to 0G after run completion · verifiable with swati verify --uri
